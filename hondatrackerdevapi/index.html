Family Car Tracker - OpenAPI 3.0

This is a sample family car tracking application allowing us to book car usage times.

Since I'm are using cookie auth on this Family Car project, the Try It Out feature won't work in browsers due to security limitations. Please use Postman to try these requests.

Database is located in AWS DynamoDB, it's named development-family-car-booking-app

get /bookings
Get all bookings by some criterion (car, user etc.) (bookingsGet)

List of bookings for a particular car or a particular user. For example, for family Honda, this should return a list of all bookings, including created by Masha, Ilya and Papa.

When userId is included in query params together with carId, then user details are also returned

Query parameters

carId (optional)
Query Parameter — User's car id (license plate)

Return type

post /login
Login with your credentials (loginPost)
You must provide a cookie called sessionId with a value that corresponds to your user in the database. We assume that the user is already registered, so you should find it in the DynamoDB database. When calling this endpoint for the first time, you should not supply any auth cookie (sessionId). But for subsequent calls, please ensure you attach the sessionId cookie returned from previous /login calls. Ask backend team for any support.
