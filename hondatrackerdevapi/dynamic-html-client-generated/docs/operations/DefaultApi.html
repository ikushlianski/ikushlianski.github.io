This is the API

bookingsGet

List of bookings for a particular car or a particular user. For example, for family Honda, this should return a list of all bookings, including created by Masha, Ilya and Papa. When `userId` is included in query params together with `carId`, then user details are also returned

URL

https://18qv4o7xm9.execute-api.eu-west-1.amazonaws.com/bookings

HTTP Method

GET

Response Type

inline_response_200_2

Parameters

  • Query: carId String

    User's car id (license plate)

  • sessionId String

    Cookie that should go with your user's request. This `sessionId` cookie value is located in DynamoDB database, assuming your user logged in previously. If you don't know your sessionId, it was likely not generated. Call the `/login` endpoint to get the session id generated and returned to you in the cookies

loginPost

You must provide a cookie called `sessionId` with a value that corresponds to your user in the database. We assume that the user is already registered, so you should find it in the DynamoDB database. When calling this endpoint for the first time, you should not supply any auth cookie (`sessionId`). But for subsequent calls, please ensure you attach the `sessionId` cookie returned from previous `/login` calls. Ask backend team for any support.
